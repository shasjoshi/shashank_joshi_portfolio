Used Car Leasing Platform

Data Engineering with Palantir Foundry

Duration: May 2023 - June 2023
Location: Mumbai, India
Role: Palantir Data Engineer

Project Overview

  • Designed and implemented a comprehensive data pipeline for a used car leasing platform, processing over 50,000 vehicle records with 95% data accuracy.
  • Developed a custom ontology model to represent complex relationships between vehicles, leasing contracts, customers, and financial data.
  • Created dynamic update/delete actions with proper authorization controls to maintain data integrity across the platform.
  • Built an interactive dashboard with real-time analytics for vehicle utilization, leasing patterns, and maintenance scheduling.
  • Implemented automated data quality checks that reduced manual validation efforts by 70%.
  • Designed a maintenance prediction system using historical data that improved vehicle uptime by 25%.
  • Integrated with external data sources including vehicle history reports and market pricing APIs.

Dashboard Overview

The main dashboard provides a comprehensive overview of key metrics related to leasing cars. It highlights critical data, such as the number of open and closed issues, categorized by priority levels (high, medium, and low). On the left side, a filter panel allows users to refine the data by model, manufacturer, and priority for a tailored view.
On the right side, the dashboard features a graph displaying a year-wise trend analysis of issues, segmented by priority. This visualization enables users to identify patterns and monitor changes over time effectively.
At the top of the dashboard, there are two tabs for easy navigation: one for the car homepage and another for the issue inbox, ensuring quick access to relevant information. Additionally, a night mode toggle is included, allowing users to customize the dashboard theme for a more comfortable viewing experience in low-light environments.

    Leasing Analytics

    The issue inbox dashboard serves as a centralized platform for managing and monitoring issues. It displays key details such as the reported date, status, priority, and a brief description of each issue. Users can quickly preview additional details by hovering over any issue. On the left-hand side, a filtering panel allows users to refine the view based on priority levels (high, medium, low), status categories (open, in progress, resolved), date ranges, and issue categories such as manufacturing, leasing, or maintenance. The dashboard also includes interactive elements that let users update issue statuses, assign responsibilities, or escalate problems directly from the interface. Additionally, a search function helps locate specific issues using keywords or IDs. This setup ensures efficient tracking and resolution of issues while keeping all stakeholders informed.

    Maintenance Tracking

    Showcasing detailed information about issues related to manufacturing or leasing for a specific car, accessible through the issue tab. When a particular issue is selected, a drawer opens on the right side, displaying comprehensive details of the selected issue.

    Reporting Module

    A pop-up triggers an action to update the status, complemented by a custom reporting interface that enables authorized users to make status updates, ensuring alignment with real-time changes.

    Ontology Model

    I have implemented the semantic layer, linking the car dataset with the issue datasets while defining business rules and applying access controls. Two key actions, "Modify Priority" and "Update Status," have been added to ensure the system stays synchronized with real-time data updates.
